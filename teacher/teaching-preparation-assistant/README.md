## Skill 简介

**teaching-preparation-assistant** 是一个面向大学计算机专业教师的备课助手 Skill。教师只需输入小节名称和简要说明（如"Python 面向对象编程，讲类和对象、继承、多态、封装"），Skill 即可自动生成一份详尽的教学内容文档。

这份文档服务于两个核心场景：

| 用途 | 使用者 | 说明 |
|------|--------|------|
| **PPT 内容底稿** | 教师 | 教师从文档中提炼要点、选取图表和代码，手工制作上课演示用的教学 PPT |
| **学生自学材料** | 学生 | 文档发布到飞书 / 语雀 / Notion 等云笔记平台后，直接分享给学生课前预习和课后复习 |

文档不是 PPT 那种提纲挈领的要点列表，而是一篇**学生可以独立阅读、跟着动手实践、学完后能真正掌握知识点的详尽讲义**。同时，文档结构清晰、层次分明，方便教师从中提取 PPT 素材。

覆盖领域：数据结构、算法、操作系统、计算机网络、数据库、编程语言、软件工程、人工智能等 CS 全领域。

## 适用场景

### 会触发此 Skill 的表述

当用户提到以下内容时自动触发：

- "帮我备课"、"教学文档"、"知识点讲解"
- "教学材料"、"讲义"、"课程文档"
- "给学生看的学习资料"、"生成一篇教学内容"
- 描述了一个计算机相关的知识点，并希望生成详细教学内容

### 使用示例

```
帮我备一节课，知识点是「快速排序」，面向大三学生，用 Python 示例
```

```
知识点：TCP 三次握手。简要说明：讲解为什么需要三次握手而不是两次、
握手过程中各字段的作用、握手失败的处理。面向计算机网络课程大二学生。
```

```
帮我生成「Python 面向对象编程」的教学文档，讲类和对象、封装、继承、多态，
生成后保存到飞书。
```

### 教师需要提供什么

| 信息 | 是否必须 | 示例 |
|------|---------|------|
| 小节名称 | **必须** | "Python 面向对象编程"、"二叉搜索树"、"TCP 三次握手" |
| 简要说明 | **必须** | 几句话描述要讲什么、侧重点、学生阶段 |
| 编程语言偏好 | 可选 | 不指定则自动选择最合适的语言 |
| 特定要求 | 可选 | "重点讲多态"、"需要和 Java 对比" |

> 教师只需给出名称和简略说明即可——备课助手的工作就是从简要信息扩展出完整文档。

## 核心设计原则

| 原则 | 说明 |
|------|------|
| **自学优先** | 假设学生没有老师在旁边讲解，文档本身要能讲清楚所有内容 |
| **由浅入深** | 从生活直觉或已有经验出发，逐步深入到原理、实现和工程实践 |
| **图文并茂** | 在关键概念处用 Mermaid 图表辅助理解，让抽象概念可视化 |
| **代码驱动** | 用真实可运行的代码示例说明概念，代码有注释和运行结果 |
| **知识串联** | 对前置知识提供参考链接，帮助学生查漏补缺；对拓展知识给出延伸阅读 |
| **PPT 友好** | 每个 `##` 章节可对应 PPT 一组幻灯片，核心要点加粗标注，图表和代码可直接截图放入 PPT |

### PPT 友好性细则

- 每个 `##` 章节对应 PPT 的一组幻灯片，章节内逻辑相对独立
- 核心要点用**加粗**或列表标注，方便教师快速提取 PPT 标题和要点
- Mermaid 图表是现成的 PPT 素材，每张图聚焦一个要点
- 代码块控制在 30 行以内，方便截图放入 PPT
- 善用对比表格——表格天然就是 PPT 的一页内容

### 自学友好元素

文档中适当位置插入以下元素，增强自学体验：

| 元素 | 用途 |
|------|------|
| `> 🤔 思考一下` | 在讲解关键概念前，先抛出问题让学生思考 |
| `> 💡 小贴士` | 补充实用技巧或注意事项 |
| `> 🏠 类比理解` | 用生活类比帮助理解抽象概念 |
| `> ⚠️ 注意` | 强调容易混淆或出错的地方 |

```

│  └── 信息不足时简短追问 1 个问题                           │
├─────────────────────────────────────────────────────────┤
│  第二步：确定文档结构                                      │
│  ├── 根据知识点类型选择模板（通用 / 算法 / 系统 / 语言 / DB）│
│  └── 确定章节安排                                         │
├─────────────────────────────────────────────────────────┤
│  第三步：生成文档内容                                      │
│  ├── 按章节逐步生成                                       │
│  ├── 插入 Mermaid 图表（流程/时序/状态/类图/ER图）          │
│  ├── 插入可运行代码（有注释 + 预期输出）                    │
│  ├── 添加前置知识链接和延伸阅读                            │
│  └── 插入自学友好元素（思考题/小贴士/类比/注意）             │
├─────────────────────────────────────────────────────────┤
│  第四步：质量自检（9 项检查清单）                           │
│  ├── 每个新概念都有解释                                    │
│  ├── 代码语法正确、可运行、有注释、有输出                    │
│  ├── 关键流程和结构有 Mermaid 图表                         │
│  ├── 前置知识有参考链接                                    │
│  ├── 有练习题或思考题                                     │
│  ├── 标准 Markdown，可直接粘贴到云笔记                     │
│  ├── 标题层级清晰                                         │
│  ├── 理论后必跟例子或图示                                  │
│  └── 章节结构便于提取 PPT 素材                             │
├─────────────────────────────────────────────────────────┤
│  第五步：保存与发布                                        │
│  ├── 方式一：保存为本地 .md 文件                           │
│  ├── 方式二：保存到飞书云文档（需一次性配置）                │
│  └── 方式三：直接输出到对话                                │
```

## 文档结构模板

### 通用结构

适用于大多数 CS 知识点：

```markdown
# {知识点名称}

## 学习目标
## 前置知识
## 为什么学这个？（现实动机）
## 核心概念讲解
### 概念 1：...
### 概念 2：...
## 代码实战
## 常见误区与易错点
## 动手练习
## 知识拓展与延伸阅读
## 本节小结
```

### 分类型结构

不同类型的知识点可灵活调整：

| 类型 | 结构特点 |
|------|---------|
| **算法类** | 增加"算法步骤详解"和"复杂度分析"章节，配流程图和代码逐行讲解 |
| **系统/协议类** | 增加"架构概览"和"交互流程"章节，配时序图和状态图 |
| **编程语言类** | 增加"语法基础"和"最佳实践"章节，配大量代码示例和类图 |
| **数据库类** | 增加"数据模型"和"查询优化"章节，配 ER 图和 SQL 示例 |

详细模板和写作指南见 `references/document-template.md`。

## 飞书云文档集成

Skill 支持将生成的 Markdown 文档自动导入飞书云文档，导入后可直接分享链接给学生。

### 认证方式说明（为什么不用账号密码）

飞书开放平台 API **不使用个人飞书账号密码**进行认证，而是使用**应用凭据**（App ID + App Secret）。

这是飞书官方的安全设计——你在飞书开放平台创建一个"自建应用"（相当于一个专用机器人），用这个应用的凭据来调用 API 操作云文档，而不是暴露你的个人登录密码。

| 概念 | 说明 | 示例 |
|------|------|------|
| **App ID** | 飞书应用的唯一标识 | `cli_xxxxxxxxxxxx` |
| **App Secret** | 飞书应用的密钥，相当于应用的"密码" | 一长串密钥字符串 |
| **folder_token** | 飞书文件夹的标识，决定文档保存到哪个文件夹 | `fldrXXXXXXXXXXXX` |

**使用应用凭据的好处：**

- 凭据可以随时重置或禁用，不影响你正常登录飞书
- 可以精确控制应用只能访问云文档，不能碰你的聊天、日历等其他数据
- 应用凭据只存在你本地配置文件中，不会上传到任何地方

### 一次性配置（5 步）

以下配置只需做一次，之后所有文档自动使用此配置。

#### 第 1 步：创建飞书自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/)，用你的飞书账号登录
2. 点击「开发者后台」→「创建企业自建应用」
3. 填写应用名称（如"教学文档助手"）和描述
4. 创建完成后，进入应用详情页

> **注意**：你需要是飞书租户的管理员，或有创建应用的权限。如果你不是管理员，请联系管理员协助创建。

#### 第 2 步：配置应用权限

在应用详情页的「权限管理」中，搜索并开通以下权限：

| 权限名称 | 权限标识 | 用途 |
|---------|---------|------|
| 查看、评论、编辑和管理云空间中所有文件 | `drive:drive` | 上传文件到云空间 |
| 上传文件 | `drive:file:upload` | 上传 Markdown 文件 |
| 查看、评论、编辑和管理文档 | `docx:document` | 创建和管理飞书文档 |

开通权限后，点击「创建版本」→「发布」（企业自建应用通常即时生效）。

#### 第 3 步：获取应用凭据

在应用详情页的「凭证与基础信息」中，找到：

- **App ID**：格式为 `cli_xxxxxxxxxxxx`
- **App Secret**：一长串密钥字符串

请妥善保存这两个值。

#### 第 4 步：获取目标文件夹的 token

决定教学文档保存到飞书的哪个文件夹，然后获取该文件夹的 token：

1. 打开飞书网页端 → 云文档
2. 进入你想存放教学文档的文件夹（如"教学资料"文件夹）
3. 查看浏览器地址栏，URL 类似：
   `https://xxx.feishu.cn/drive/folder/fldrXXXXXXXXXXXX`
4. URL 最后一段（`fldrXXXXXXXXXXXX`）就是 folder_token

> 如果不设置 folder_token，文档会创建在应用的默认空间中。建议设置一个专用文件夹，方便管理和分享。

#### 第 5 步：创建并填写配置文件

运行以下命令，从模板创建配置文件：

```bash
python scripts/save_to_feishu.py --init
```

这会在 `scripts/` 目录下创建 `feishu_config.json` 文件。打开它，填入你的凭据：

```json
{
    "app_id": "cli_axxxxxxxxxxxx",
    "app_secret": "你的App Secret",
    "folder_token": "fldrXXXXXXXXXXXX"
}
```

> 只需修改 `app_id`、`app_secret`、`folder_token` 三个字段的值。以 `_` 开头的字段是说明注释，不用管。

#### 验证配置

填好凭据后，运行验证命令确认配置正确：

```bash
python scripts/save_to_feishu.py --check
```

如果输出"配置验证通过"，说明一切就绪，可以正常使用了。

如果验证失败，请根据错误提示检查：
- app_id / app_secret 是否填写正确
- 应用是否已发布
- 应用权限是否已开通

### 日常使用（配置完成后）

配置完成后，每次生成教学文档后只需一条命令：

```bash
python scripts/save_to_feishu.py --file "Python面向对象编程.md" --name "Python面向对象编程"
```

脚本会自动：
1. 读取已保存的配置（app_id、app_secret、folder_token）
2. 上传 Markdown 文件到飞书
3. 转换为飞书文档
4. 保存到你在配置中指定的文件夹
5. 返回飞书文档链接

### 脚本命令速查

| 命令 | 用途 |
|------|------|
| `--init` | 从模板创建配置文件，引导教师填写 |
| `--check` | 验证配置是否正确（检查字段 + 实际连接飞书 API 测试） |
| `--file <路径>` | 指定要导入的 Markdown 文件路径（必填） |
| `--name <标题>` | 指定飞书文档标题（必填） |
| `--folder <token>` | 临时指定文件夹 token，覆盖配置文件中的设置（可选） |
| `--config <路径>` | 指定其他配置文件路径（可选） |

#### 临时切换文件夹

如果想把某次生成的文档保存到不同的文件夹，不需要修改配置文件：

```bash
python scripts/save_to_feishu.py --file "实验指导.md" --name "实验指导" --folder fldrOtherFolderXXXX
```

### 手动导入替代方案

如果不想配置飞书 API（比如你不是管理员无法创建应用），也可以手动导入：

#### 方法一：飞书云空间导入

1. 打开飞书云空间（飞书网页端 → 云文档）
2. 进入目标文件夹，点击右上角「+」→「导入」
3. 选择「Markdown」格式
4. 上传 `.md` 文件
5. 飞书会自动将 Markdown 转换为飞书文档

#### 方法二：复制粘贴

1. 用 Markdown 编辑器（如 Typora）打开 `.md` 文件
2. 全选复制
3. 在飞书文档中粘贴
4. 飞书会自动识别 Markdown 格式（包括标题、代码块、表格等）

> **注意**：飞书的 Mermaid 图表渲染需要在新版飞书文档（docx）中才支持。如果粘贴后 Mermaid 代码块没有渲染为图表，可能需要手动插入图表。

### 常见问题

**Q: 我不是飞书管理员，没法创建应用怎么办？**

可以请管理员帮你创建应用并开通权限，然后把 App ID 和 App Secret 给你。或者使用「手动导入替代方案」，不需要 API 配置。

**Q: 配置文件保存在哪里？安全吗？**

配置文件保存在 `scripts/feishu_config.json`，只存在于你的本地电脑上。脚本不会将凭据上传到任何地方。建议：
- 不要把 `feishu_config.json` 提交到 Git 仓库
- 如果担心安全，可以将文件权限设置为仅自己可读

**Q: App Secret 忘了怎么办？**

在飞书开放平台 → 应用详情 → 凭证与基础信息中，可以重置 App Secret。重置后需要同步更新配置文件中的 `app_secret` 字段。

**Q: 导入报错 "permission denied" 怎么办？**

确保：
1. 应用已开通所需权限（见第 2 步）
2. 应用已发布版本
3. 如果指定了 folder_token，确保应用有该文件夹的访问权限

**Q: 导入后 Mermaid 图表没有渲染怎么办？**

飞书文档支持 Mermaid 代码块渲染。如果导入后图表没有显示：
1. 确认使用的是新版飞书文档（docx 格式）
2. 点击代码块右上角的语言选择器，确保设置为 "mermaid"
3. 如果仍未渲染，可以在飞书文档中手动插入「流程图」组件

**Q: 想换一个文件夹保存文档怎么办？**

两种方式：
- **临时更换**：命令行加 `--folder` 参数，不影响配置文件
- **永久更换**：编辑 `feishu_config.json` 中的 `folder_token` 字段

**Q: 如何批量导入多个文档？**

```bash
for %f in (*.md) do python scripts/save_to_feishu.py --file "%f" --name "%~nf"
```

或者让 AI 助手帮你批量执行。

### 飞书导入脚本技术说明

脚本 `save_to_feishu.py` 使用 Python 标准库（`urllib`），无需安装任何第三方依赖。工作流程：

```
读取配置 → 获取 tenant_access_token → 上传 .md 文件 → 创建导入任务 → 轮询导入结果 → 返回文档链接
```

涉及三个飞书 API：
1. `auth/v3/tenant_access_token/internal` — 获取访问令牌
2. `drive/v1/files/upload_all` — 上传文件
3. `drive/v1/import_tasks` — 创建导入任务并轮询结果
