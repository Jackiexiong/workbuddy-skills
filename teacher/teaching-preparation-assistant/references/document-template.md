# 教学文档模板与写作指南

本文件定义了教学文档的完整结构和每个章节的写作要求。根据知识点类型灵活选用。

## 文档的两大用途

这份文档服务于两个场景，写作时需要同时兼顾：

1. **教师制作 PPT 的内容底稿**：教师拿到文档后，从中提炼要点、选取图表和代码，制作上课演示用的 PPT
2. **学生的补充学习材料**：文档发布到飞书/语雀等云笔记平台后，直接分享给学生自学阅读

因此，文档既要有 PPT 所需的清晰结构和可提取的要点，又要有学生自学所需的详尽讲解。

### PPT 友好性原则

- 每个 `##` 章节对应 PPT 的一组幻灯片，章节内逻辑应相对独立
- 核心要点用**加粗**或列表标注，方便教师快速提取 PPT 标题和要点
- Mermaid 图表是现成的 PPT 素材，每张图聚焦一个要点
- 代码块控制在 30 行以内，方便截图放入 PPT
- 善用对比表格——表格天然就是 PPT 的一页内容

## 各章节写作指南

### 学习目标

用 3-5 条要点告诉学生"学完这节后你能做什么"。用动词开头，描述可验证的能力。

**好的写法：**
```
学完本节后，你能够：
- 用自己的语言解释什么是虚拟内存以及它解决了什么问题
- 画出页表、TLB、缺页中断的工作流程图
- 手动计算给定页面大小和虚拟地址的页号和页内偏移
- 分析缺页频繁时系统性能下降的原因
```

**避免：**
```
学完本节后你将了解虚拟内存的相关知识。  ← 太模糊，"了解"不可验证
```

---

### 前置知识

列出学习本节需要先掌握的知识，每条配上参考链接。目的是让学生如果发现某项前置知识不扎实，可以跳过去补课。

**写法示例：**
```markdown
## 前置知识

学习本节前，建议你已掌握以下内容：

| 前置知识 | 说明 | 参考链接 |
|---------|------|---------|
| 进程与地址空间 | 理解程序运行时的内存布局 | [操作系统——进程管理](https://example.com/os-process) |
| 二进制与十六进制 | 能进行基本的进制转换 | [计算机基础——进制转换](https://example.com/binary) |
| CPU 缓存基础 | 了解 L1/L2/L3 缓存层次 | [CPU 缓存原理详解](https://example.com/cpu-cache) |

如果以上某些内容还不熟悉，没关系，先点链接花 10-15 分钟了解一下，再回来学习本节。
```

用表格让前置知识一目了然，链接描述要说明"点进去能看到什么"。

### 为什么学这个？

用 1-2 段话讲清楚这个知识点解决什么现实问题。从学生的经验出发，建立学习动机。

**写法要点：**
- 先描述一个学生能感知到的问题或现象
- 再引出这个知识点是如何解决它的
- 可以用"如果没有 X，会怎样？"的反向提问来制造张力

**示例：**
```markdown
## 为什么学这个？

你有没有想过：你的电脑只有 8GB 内存，但你同时开着浏览器（几十个标签页）、
VS Code、Spotify、微信——加起来明明要用掉 20GB+ 的内存，为什么系统没有崩溃？

答案就是**虚拟内存**。它是现代操作系统最精妙的设计之一，让每个程序都以为自己
独占了整个内存空间，而实际物理内存只有那么大。没有虚拟内存，就没有现代多任务
操作系统——你不可能同时运行这么多程序。

今天我们就来拆解这个"魔法"是怎么实现的。
```

### 核心概念讲解

这是文档的主体。根据知识点拆分为多个子概念，每个子概念按以下模式展开：

```
1. 概念引入（这是什么？为什么需要它？）
2. 详细讲解（原理、机制、规则）
3. 图表演示（用 Mermaid 可视化）
4. 代码或实例（用具体例子验证理解）
5. 一句话小结
```

每个子概念之间要有逻辑过渡，让学生感受到知识是层层递进而非跳跃的。

**关键原则：**
- 每引入一个新术语，立即用一句话解释它，不要假设学生知道
- 讲完抽象原理后，立刻用一个具体例子落地
- 如果一个概念可以用图说明，优先用图，文字作为补充

### 代码实战

如果核心概念中已经穿插了代码片段，这个章节可以放一个更完整的、综合性的代码示例——把前面讲的概念串起来。

**写法要点：**
- 代码不要太长（控制在 50-80 行以内），如果太长就拆成多段讲解
- 代码前面用一段话说明"这段代码做了什么"
- 关键行有注释，解释设计意图
- 代码后面给出预期运行结果
- 如果有常见的错误写法，可以展示"错误写法 vs 正确写法"对比

**示例格式：**
```markdown
下面用 Python 实现一个简单的二叉搜索树，包含插入和查找操作：

​```python
class BSTNode:
    def __init__(self, val):
        self.val = val
        self.left = None   # 左子树存比当前节点小的值
        self.right = None  # 右子树存比当前节点大的值

class BST:
    def __init__(self):
        self.root = None

    def insert(self, val):
        if not self.root:
            self.root = BSTNode(val)
            return
        cur = self.root
        while True:
            if val < cur.val:
                if cur.left is None:
                    cur.left = BSTNode(val)  # 找到空位，插入
                    return
                cur = cur.left
            else:
                if cur.right is None:
                    cur.right = BSTNode(val)
                    return
                cur = cur.right
​```

运行结果：
​```
>>> tree = BST()
>>> for v in [50, 30, 70, 20, 40]:
...     tree.insert(v)
>>> # 此时树的结构：
>>> #       50
>>> #      /  \
>>> #    30    70
>>> #   /  \
>>> #  20   40
​```
```

### 常见误区与易错点

列出学生在学习这个知识点时最容易犯的 2-4 个错误。每个误区格式：

```
**误区：** {错误理解}
**正确理解：** {正确理解}
**为什么：** {简要解释原因}
```

这个章节非常有价值，因为自学时没有老师纠错，提前预警可以帮学生少走弯路。

### 动手练习

设计 2-4 道练习题，从易到难：

- **基础题**：直接应用本节概念（如"手动模拟某过程"）
- **进阶题**：需要理解原理才能回答（如"分析某种情况下的行为"）
- **挑战题**：需要综合运用或动手编程（可选）

不一定要给出答案——可以说"答案下节课公布"或"把你的答案发到课程群里讨论"。但基础题建议附上参考答案或提示。

提供 2-4 个深入学习的资源链接，每条说明"这个资源里有什么，适合什么阶段看"。

```markdown
## 知识拓展与延伸阅读

### 本节小结

用 3-5 条要点总结本节核心内容。用加粗关键词让学生快速回顾。

```markdown
## 本节小结

- **虚拟内存**让每个进程拥有独立的、连续的地址空间，实际物理内存由操作系统统一管理
- **页表**实现虚拟地址到物理地址的映射，**TLB** 是页表的硬件缓存，加速地址翻译
- **缺页中断**在访问未映射的页面时触发，操作系统负责从磁盘加载页面
- **页面置换算法**（如 LRU）决定内存不足时换出哪个页面
```

## 完整示例片段

以下是一个"快速排序"知识点的核心概念讲解片段，展示写作风格和图表配合：

```markdown
### 核心思想：分而治之

快速排序的核心思想用一句话概括：**选一个数当"基准"，把比它小的放左边，比它大的放右边，然后对左右两边重复这个过程。**

来看一个具体例子。假设我们要排序 `[5, 3, 8, 1, 9, 2, 7]`，选第一个元素 `5` 作为基准：

​```mermaid
flowchart TD
    A["原始数组: [5, 3, 8, 1, 9, 2, 7]"] --> B["选基准 pivot = 5"]
    B --> C["分区: 比5小的移左边, 比5大的移右边"]
    C --> D["[3, 1, 2]  5  [8, 9, 7]"]
    D --> E["对左边重复 → 排序 [3, 1, 2]"]
    D --> F["对右边重复 → 排序 [8, 9, 7]"]
    E --> G["[1, 2, 3]  5  [7, 8, 9]"]
    F --> G
    G --> H["最终结果: [1, 2, 3, 5, 7, 8, 9]"]
​```

> **类比理解**：想象你在整理书架上的书。你随手拿起一本书作为"标准"，
> 把比它薄的放左边，比它厚的放右边。然后你对左右两堆分别再做同样的事。
> 最终整个书架就排好序了。这就是"分而治之"——把大问题拆成小问题，小问题解决了，大问题也就解决了。

这个"分区"（partition）操作是快速排序的灵魂，下面我们详细拆解它怎么实现。
```
