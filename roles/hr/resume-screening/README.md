# 📄 简历筛选技能 — 使用说明

> 适用于 **WorkBuddy / CodeBuddy** 的 HR 简历筛选利器。  
> 从招聘网站批量下载的简历（docx / pdf / html），交给 AI 帮你智能筛选、评分、排名，一键挑出最合适的候选人。

---

## 🎯 它能做什么

把几十上百份简历和一个岗位的职位要求（JD）告诉我，我就能：

| 功能 | 说明 |
|------|------|
| 🔍 **批量扫描** | 自动遍历文件夹，识别所有简历文件 |
| 📖 **智能解析** | 支持 `.docx` / `.pdf` / `.html` / `.txt` 四种格式的内容提取 |
| 📊 **多维评分** | 从 6 个维度对照 JD 逐份评分（含硬性条件一票否决） |
| 🏆 **自动排名** | 按匹配度排序，选出 Top N |
| 📁 **结果归档** | 将选定简历复制到新文件夹，方便后续操作 |
| 📋 **输出报告** | 在对话中展示完整排名报告，可选导出 CSV |

---

## 🚀 快速上手

### 你只需要提供

1. **简历文件夹路径** — 存放简历的目录（支持嵌套子目录）
2. **岗位 JD 文本** — 直接粘贴职位要求全文
3. **期望选出份数**（可选，默认 10 份）

### 说一句话就行

> "帮我筛选一下 `D:\招聘\Java高级工程师` 这个文件夹里的简历，JD 是：本科以上，3 年以上 Java 开发经验，熟悉 Spring Boot 和微服务，有电商经验优先。挑出最合适的 10 份。"

我自动完成扫描 → 解析 → 评分 → 排名 → 归档 → 出报告的全流程。

---

## 📂 支持的简历格式

| 格式 | 来源场景 | 解析方式 |
|------|----------|----------|
| `.docx` | 招聘网站下载 / 候选人投递 | python-docx |
| `.pdf`（文字型） | 招聘网站导出 / 候选人投递 | pypdf / pdfplumber |
| `.pdf`（扫描型） | 纸质简历扫描件 | ❌ 需手动转为文字 |
| `.html` | BOSS直聘/猎聘/智联等网站导出 | BeautifulSoup |
| `.txt` | 纯文本简历 | 直接读取 |

> 💡 如果电脑上还没有安装解析库，我引导你一键安装：
> ```bash
> pip install python-docx pypdf pdfplumber beautifulsoup4 chardet
> ```

## 📊 评分维度

每份简历会从以下 **6 个维度** 进行评分：

| 维度 | 权重 | 评什么 |
|------|------|--------|
| 🎯 **硬性条件匹配** | 30% | 学历、专业、工作年限、证书等 — ❌ 不达标直接淘汰 |
| 🛠 **技能栈匹配** | 25% | JD 要求的技能栈掌握程度 |
| 📁 **项目/经验匹配** | 20% | 过往项目经验与岗位职责的吻合度 |
| 💬 **软性素质** | 10% | 沟通、协作、管理能力 |
| 🏢 **行业/公司背景** | 10% | 相关行业经验、知名公司经历 |
| ⭐ **简历质量** | 5% | 简历清晰度、信息完整度、量化产出 |

## 💬 使用示例

### 标准筛选

```
你：帮我筛选 D:\简历库\前端 里的简历，JD 是：本科以上，3 年前端经验，
    熟悉 Vue3/React，了解 Node.js，有移动端 H5 经验优先。挑 10 份。
```

### 追加条件

```
你：在上次的结果里，只要 985/211 毕业的。
我：好的，我重新筛选（或从上次结果中过滤）。
```

### 海量简历（200+）

```
你：这里有 200 多份，先粗筛淘汰明显不合适的，剩下的再精细评分。
我：好的！第一轮先做硬性条件过滤，第二轮精细评分排名。
```

### 导出排名表

```
你：把排名表导出来，我要发邮件。
我：运行导出命令，生成 CSV 文件。
```

## ❓ 常见问题

| 问题 | 解答 |
|------|------|
| **简历可以放在子文件夹里吗？** | 可以。脚本会递归遍历所有子目录。 |
| **PDF 提取不出来文字怎么办？** | 可能是扫描件（图片），我会提示你如何转换成文字。 |
| **HTML 简历排版乱怎么办？** | 我会自动提取可见文本，排除导航/广告等噪音。 |
| **能筛选多少份？** | 不限数量。200 份以内单轮完成，200+ 建议先粗筛再精筛。 |
| **重复投递的简历会重复计算吗？** | 会识别并去重，只保留一份评分。 |
| **结果文件夹在哪里？** | 在简历文件夹下生成 `_筛选结果_TopN/` 目录，原始文件不改动。 |
| **排名表可以导出吗？** | 可以。我可以生成 CSV 文件，可在 Excel 中打开。 |

---

## 📝 注意事项

- 技能本身**不修改原始文件**，只读取和复制
- 评分标准可调整 — 你可以随时说"把 XX 维度权重调高"或"更看重项目经验"
- 自动跳过隐藏文件和临时文件（如 `~$` 开头的临时文档）
- 文件名不需要包含候选人姓名 — 我会从简历内容中自动识别
