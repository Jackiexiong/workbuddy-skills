# 💻 课程设计代码批量评分技能

> 适用于 **WorkBuddy / CodeBuddy** 的教师评分利器。  
> 输入学生信息表和设计任务书，自动拉取每位学生的 Git 仓库代码，智能评分并回写结果。

---

## 🎯 它能解决什么问题

**传统方式**：老师需要手动打开每个学生的 Git 仓库 → 下载代码 → 打开 IDE 审查 → 对照任务书打分 → 录入 Excel

**有了这个技能后，你只需要说一句：**

> "帮我对软件工程课设的代码进行评分。学生表在 `D:/课设/学生信息表.xlsx`，任务书在 `D:/课设/任务书.docx`，Git 账号是 `admin/abc123`。"

后面的全部由 AI 自动完成：

| 步骤 | 自动完成 |
|------|---------|
| 📂 读取学生表 | 解析所有学生信息和仓库地址 |
| 📖 解析任务书 | 提取功能需求和评分要点 |
| 📥 拉取代码 | 逐位学生 clone Git 仓库（浅克隆，速度快） |
| 🔍 审查代码 | 检查功能实现、代码质量、技术正确性 |
| 📊 5 维评分 | 功能完整性(40%) + 代码质量(25%) + 技术正确性(15%) + 项目结构(10%) + 创新拓展(10%) |
| ✍️ 写回 Excel | 分数和评语自动填入原表 |
| 📋 输出报告 | 统计摘要、等级分布、常见问题 Top3 |

---

## 📋 输入要求

### 学生信息 Excel

表头需包含以下列（列名不强制完全一致，技能会自动匹配）：

| 必填列 | 说明 |
|--------|------|
| **学号** | 学生的唯一标识，用于匹配评分结果 |
| **班级** | 学生所在班级 |
| **姓名** | 学生姓名 |
| **代码仓库地址** | Git 仓库 URL（如 https://github.com/student/repo） |
| **分数** | 可空，评分后自动填充 |
| **备注** | 可空，评分后自动填充评语或异常说明 |

### 课程设计任务书（Word）

任务书需要包含以下信息（技能会自动提取）：

- 🎯 **项目背景**：业务场景描述
- 📝 **功能需求清单**：必须实现的功能列表
- ⚙️ **技术/架构要求**：指定语言、框架、数据库等
- 📐 **非功能要求**：性能、安全等
- 📦 **交付物要求**：代码结构、文档等

### Git 凭证

- **用户名** 和 **密码**（用于 URL 嵌入认证）
  - 格式：`https://用户名:密码@github.com/学生/仓库.git`

> 🔒 凭证只在当前会话中使用，不会永久存储。

## 🏆 5 维度评分体系

| 维度 | 权重 | 评什么 |
|------|------|--------|
| 🎯 **功能完整性** | **40%** | 对照任务书功能清单，逐项检查实现完成度 |
| 📐 **代码质量** | **25%** | 命名规范、注释完整度、模块化、健壮性 |
| ⚙️ **技术正确性** | **15%** | 技术方案是否合理、有无 Bug 或安全隐患 |
| 🏗️ **项目结构与工程化** | **10%** | 目录结构、构建工具、依赖管理、版本控制 |
| 💡 **创新与拓展** | **10%** | 额外功能、测试覆盖、性能优化、技术亮点 |

### 等级映射

| 等级 | 分数 | 含义 |
|------|------|------|
| 🥇 优秀 | 90-100 | 功能完整，代码质量高，有额外亮点 |
| 🥈 良好 | 80-89 | 功能基本完整，代码规范 |
| 🥉 中等 | 70-79 | 主要功能实现，代码质量一般 |
| 📋 及格 | 60-69 | 部分功能实现 |
| ❌ 不及格 | <60 | 核心功能未实现 |
| ⚠️ 异常 | N/A | 仓库无法访问或为空 |

## 💬 使用示例

### 基本用法

```
你：帮我对数据库课设的代码评分。
    学生表：D:/课程/学生信息表.xlsx
    任务书：D:/课程/任务书.docx
    Git 账号：admin / pass123
    挑最合适的 10 份。

我：开始处理...
    [1/32] 张三 — ✅ 评分完成 (85分)
    [2/32] 李四 — ✅ 评分完成 (92分)
    [3/32] 王五 — ⚠️ 仓库无法访问 (404)
    ...
    [32/32] 赵六 — ✅ 评分完成 (78分)
    
    全部完成！Excel 已更新。
```

### 追加评分

```
你：上次有 3 位补交的同学加到表里了，帮我也评了。

我：好的，我检查到学号 2024033、2024034、2024035 的分数列为空。
    开始处理这 3 位...
```

### 仅查看统计

```
你：帮我看看上次评分的统计结果。

我：运行 report 模式，输出统计摘要。
```

### 清理工作目录

```
你：评完了，帮我把工作目录清理掉。

我：rm -rf ./_code_review_workspace/ ✅ 已清理
```

## ❓ 常见问题

| 问题 | 解答 |
|------|------|
| **支持哪些编程语言？** | Java、Python、JavaScript/TypeScript、C/C++、Go 等主流语言都支持 |
| **能同时评多少学生？** | 不限数量。50 人以内一次性完成，50+ 分批处理 |
| **仓库访问失败怎么办？** | 自动跳过并记录异常，不中断整体流程。最后提供异常清单 |
| **Excel 表头列名不一致怎么办？** | 自动按列名含义匹配（如"学号""Student ID""id"都可以识别） |
| **会修改原始代码吗？** | 不会。只读取代码，不做任何修改。克隆到独立工作目录 |
| **密码安全吗？** | 密码只用于本次会话的 Git 操作，不会长期存储 |
| **可以自定义评分权重吗？** | 可以。随时告诉 AI "把功能完整性权重降到30%，代码质量提到35%" |
| **任务书是 PDF 可以吗？** | 如果需要，AI 也能解析 PDF 格式的任务书 |
