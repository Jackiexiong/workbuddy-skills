# 课程设计代码批量评分技能

根据任务书的编程需求，自动拉取学生 Git 仓库代码，进行多维评分，并将结果写回 Excel 表。

## 工作流程概览

```
教师提供：学生信息 Excel + 任务书 Word + Git 凭证 + 本地工作目录
    │
    ▼
第 1 步：读取学生信息表（Excel），解析学号/姓名/仓库地址
第 2 步：读取任务书（Word），提取编程需求与评分要点
第 3 步：逐条处理每位学生：
    ├── 3.1 Git clone 代码到本地
    ├── 3.2 浏览项目结构，理解代码组织
    ├── 3.3 对照任务书需求逐项检查功能实现
    ├── 3.4 审查代码质量（结构/规范/注释/健壮性）
    ├── 3.5 进行 5 维度评分
    └── 3.6 生成评语
第 4 步：将评分结果回写到 Excel 表
第 5 步：输出统计摘要
```

## 输入

1. **学生信息 Excel 文件路径**（必填）
   - 表头需包含：`学号`、`班级`、`姓名`、`代码仓库地址`、`分数`、`备注`
   - 其中 `代码仓库地址` 必须已填写；`分数` 和 `备注` 列可为空
2. **课程设计任务书 Word 文件路径**（必填）
   - 包含项目背景、功能需求清单、技术/架构要求、交付要求
3. **Git 访问凭证**（必填）
   - 用户名和密码（用于在 URL 中嵌入认证，如 `https://user:pass@github.com/...`）
4. **本地工作目录**（可选，默认 `./_code_review_workspace/`）
   - 用于暂存克隆的代码，评审完成后可选择保留或清理

## 输出

- 直接修改原 Excel 文件，填充 `分数` 和 `备注` 列
- 在对话中展示：
  - 每位学生的评分明细（5 维度分 + 总分 + 评语）
  - 统计摘要（平均分、等级分布、常见问题 Top3）
  - 异常清单（仓库无法访问 / 代码为空 / 未提交等）

## 流程

### 第 1 步：读取学生信息表

使用 `python` 读取 Excel 文件，获取学生列表：

```bash
pip install openpyxl
python -c "
import openpyxl
wb = openpyxl.load_workbook('学生信息表.xlsx')
ws = wb.active
# 假设第 1 行为表头
headers = [cell.value for cell in ws[1]]
print('表头:', headers)
# 从第 2 行开始读数据
for row in ws.iter_rows(min_row=2, values_only=True):
    student_id, class_name, name, repo_url, score, note = row
    print(f'{student_id} | {name} | {repo_url}')
"
```

> **注意**：如果表头的列名顺序与标准不同（如"学号"在第 3 列），AI 应自动按列名匹配而非按位置。

### 第 2 步：读取任务书

使用 `python-docx` 解析任务书，提取关键信息：

```bash
pip install python-docx
python -c "
from docx import Document
doc = Document('任务书.docx')
for para in doc.paragraphs:
    if para.text.strip():
        print(para.text)
"
```

从任务书中提取以下信息结构：

| 信息项 | 说明 |
|--------|------|
| **项目背景** | 项目的业务场景和定位 |
| **功能需求清单** | 必须实现的功能模块清单（最重要） |
| **技术/架构要求** | 指定语言、框架、数据库、架构模式等 |
| **非功能要求** | 性能、安全、可用性等要求 |
| **交付物要求** | 代码结构、文档、数据库脚本等 |
| **评分标准** | 任务书中自带的评分标准（如有） |

### 第 3 步：逐条处理每位学生

对每位学生按以下子步骤执行：

#### 3.1 Git clone 代码

使用嵌入凭证的 URL 克隆代码：

```bash
# 将仓库地址中的 https:// 替换为 https://user:password@
REPO_URL="https://user:password@github.com/student/repo.git"
git clone --depth 1 "$REPO_URL" "工作目录/学号_姓名"
```

- 使用 `--depth 1` 浅克隆，加快速度
- 如 clone 失败（仓库不存在 / 无权限 / 网络问题），在备注中记录异常原因，跳到下一位学生
- 如 clone 成功但仓库为空，备注记录"仓库为空"

#### 3.2 浏览项目结构

```bash
# 查看项目目录结构（忽略 .git 目录）
cd "工作目录/学号_姓名"
find . -not -path './.git/*' -not -name '.git' | head -80
```

根据项目结构判断：
- 项目组织是否合理（分层、模块化）
- 构建工具使用情况（Maven/Gradle/npm/CMake 等）
- 配置文件是否存在（application.yml/.env/requirements.txt 等）

#### 3.3 逐项检查功能实现

对照任务书功能需求清单，检查代码中是否实现了每项功能：

```bash
# 搜索关键功能的代码实现
grep -r "关键类名/方法名" --include="*.java" --include="*.py" . | head -20
```

#### 3.4 代码质量审查

从以下维度审查代码文件（重点审查核心业务代码）：

- **代码规范性**：命名是否规范、格式是否统一
- **注释完整度**：关键逻辑是否有注释
- **健壮性**：是否有异常处理、输入验证
- **模块化**：是否合理拆分类/函数
- **代码重复度**：有无大量重复代码

#### 3.5 5 维度评分

从以下 **5 个维度** 进行评分（满分 100 分）：

| 维度 | 权重 | 评分要点 |
|------|------|----------|
| **功能完整性** | 40% | 对照任务书功能需求清单，逐项检查实现完成度 |
| **代码质量** | 25% | 代码规范性、可读性、模块化、注释质量 |
| **技术正确性** | 15% | 技术方案是否合理、是否有明显 Bug |
| **项目结构与工程化** | 10% | 目录结构、构建工具、配置管理、版本控制 |
| **创新与拓展** | 10% | 超出基本要求的额外功能、优化、技术亮点 |

**评分原则：**

- **功能完整性为基线**：如果核心功能未实现，总分上限不超过 60 分
- **每个维度给出 0-10 分**，按权重加权计算总分
- **评分必须有具体依据**：引用代码中的具体文件/行来佐证评分
- **发现重大 Bug 或安全隐患**：在备注中特别标注

详细评分量规见 `references/code-review-rubric.md`。

#### 3.6 生成评语

每位学生生成 100-200 字综合评语，包含：
- 总体评价（1 句话）
- 主要优点（1-2 个亮点，引用代码佐证）
- 不足与改进建议（1-2 个改进点）
- 鼓励语（1 句话）

### 第 4 步：回写评分结果到 Excel

使用辅助脚本将评分结果写入原 Excel 文件：

```bash
python "{baseDir}/scripts/update_scores.py" \
  --file "学生信息表.xlsx" \
  --scores '[
    {"学号": "2024001", "分数": 85, "备注": "功能完整，代码规范，建议加强异常处理"},
    {"学号": "2024002", "分数": 92, "备注": "实现优秀，有额外功能拓展"}
  ]'
```

脚本会自动：
1. 打开原 Excel 文件
2. 按 `学号` 匹配行
3. 在 `分数` 和 `备注` 列填入对应值
4. 保存文件

## 评分等级映射

| 等级 | 分数区间 | 含义 |
|------|---------|------|
| 优秀 | 90–100 | 功能完整实现，代码质量高，有额外亮点 |
| 良好 | 80–89 | 功能基本完整，代码规范，质量良好 |
| 中等 | 70–79 | 主要功能实现，代码质量一般 |
| 及格 | 60–69 | 部分功能实现，代码质量有待提高 |
| 不及格 | <60 | 核心功能未实现或代码质量严重不足 |
| 异常 | N/A | 仓库无法访问/为空/非代码仓库 |

## 示例

### 示例 1：完整评分流程

**用户说：** "帮我对软件工程课设的代码进行评分。学生表在 `D:/课设/学生信息表.xlsx`，任务书在 `D:/课设/任务书.docx`，Git 账号是 `admin/abc123`。"

**你的操作：**

1. 读取 Excel 和任务书
2. 逐位学生 clone 代码 → 审查 → 评分
3. 回写分数到 Excel
4. 输出统计报告

### 示例 2：追加评分或重新评分

**用户说：** "上次评分后又有 3 位补交的学生加到了表里，帮我也评了。"

**你的操作：**

1. 读取 Excel，识别 `分数` 列为空的学生
2. 仅对空值学生执行评分
3. 更新 Excel，输出新增学生的评分结果

### 示例 3：清理工作目录

**用户说：** "评完了，把那个工作目录删掉吧。"

**你的操作：**

```bash
rm -rf ./_code_review_workspace/
```

## 故障排除

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Git clone 失败 | 仓库 URL 错误 / 无权限 / 网络 | 记录异常，跳到下一位，不中断流程 |
| 仓库为空 | 学生未提交代码 | 备注"未提交代码"，分数留空 |
| 仓库不是代码项目 | 误上传了其他文件 | 尝试识别，确实不是则备注异常 |
| Python 库未安装 | `openpyxl` / `python-docx` 缺失 | `pip install openpyxl python-docx` |
| Excel 表头不匹配 | 列名不是预期的"学号" | 按实际列名自动匹配，不限制列名格式 |
| 代码量极大（上千文件） | 大型项目 | 重点审查核心业务代码，忽略第三方库和生成代码 |
| 仓库中含有二进制文件 | 数据库文件 / 图片等 | 自动跳过非文本文件 |
