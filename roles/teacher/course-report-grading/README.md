> 批量读取学生的课程设计/实验报告 Word 文档，对照任务书要求和评分量规逐一打分，

## Skill 简介

**course-report-grading** 是面向大学计算机专业教师的课设/实验报告批改助手。教师只需提供：

- **任务书**（Word 格式，含题目要求和评分标准）
- **学生报告文件夹**（内含多篇 Word 报告，按 `班级_姓名_学号.docx` 命名）

Skill 会逐一读取每篇报告，对照任务书要求按三大题评分，生成 150-200 字综合评语，最终输出成绩汇总 Excel 表。

当前已完整支持的课程：**《数据库应用课程设计》**。其他课程（软件工程、软件测试等）可按本文档[第 6 节](#如何适应其他课程)扩展。

### 核心能力

| 能力 | 说明 |
|------|------|
| 任务书解析 | 读取 Word 任务书，提取题目要求、检查点、评分标准 |
| 报告批量读取 | 扫描文件夹，按文件名规范解析学号/姓名/班级 |
| 智能评分 | 按三大题逐题打分，5 个考察要点融入评价 |
| 评语生成 | 每篇 150-200 字综合评语，引用具体问题 |
| Excel 汇总 | 三表输出：成绩明细 + 统计摘要 + 异常清单 |

## 评分体系

### 评分结构（忠实于任务书）

按任务书三大题给分，5 个考察要点融入打分逻辑与评语：

| 题目 | 分值 | 说明 |
|------|------|------|
| 题目1 数据库建模分析设计题 | 50 | 需求分析→ER图→关系模式→建表SQL |
| 题目2 数据库管理和优化实验 | 30 | 备份+恢复+大数据量查询优化 |
| 题目3 数据库应用开发 | 20 | 存储过程实现数据加工处理 |
| **合计** | **100** | |

### 任务书的 5 个考察要点

1. **设计正确性**：需求理解、建模正确性、规范化、可扩展性/性能/安全性
2. **软件架构与代码质量**（有编程要求时生效）：代码规范性、结构清晰度、容错性
3. **报告撰写质量**：格式遵循、内容准确性/完整性/逻辑/文字
4. **优化与创新性**：新技术使用、工程方法创新
5. **工作量与难度**：难度微调（±3 分内）

> 前 2 个要点对应三大题的技术评分；后 3 个要点影响每题得分并在综合评语中体现。

### 等级映射（五级制）

| 等级 | 分数区间 |
|------|---------|
| 优秀 | 90–100 |
| 良好 | 80–89 |
| 中等 | 70–79 |
| 及格 | 60–69 |
| 不及格 | <60 |

### 固定指标 vs 任务相关指标

| 类型 | 说明 | 每年是否变化 |
|------|------|------------|
| **固定指标** | 三大题分值占比（50/30/20）、5 个考察要点 | 不变 |
| **任务相关指标** | 每题内的具体题目内容（业务场景、查询题、存储过程题） | 每年微调 |

详细评分标准见 `references/rubric-database.md`。

## 使用步骤

### 步骤 1：准备任务书和学生报告

1. **任务书**：确保任务书为 Word（`.docx`）格式，包含三大题要求和评分标准
2. **学生报告文件夹**：将所有学生报告放在同一文件夹下，每个文件按规范命名：
   ```
   班级_姓名_学号.docx
   ```
   示例：`软件2301_陈小鹏_120231080315.docx`

> ⚠️ 文件名格式不符的文件将被拒绝并记入异常清单，不参与评分。

### 步骤 2：发起批改请求

在对话中告知任务书路径和学生报告文件夹路径：

```
帮我批改数据库应用课程设计的报告。
任务书在：D:/课程/数据库课设任务书.docx
学生报告在：D:/课程/数据库课设报告/
```

### 步骤 3：Skill 自动执行（5 步工作流程）

Skill 内部按以下流程运行：

```
┌─────────────────────────────────────────────────────┐
│ 第一步：解析任务书                                     │
│  ├── 读取 Word 任务书                                  │
│  ├── 提取：三大题要求、检查点、评分标准                │
│  └── 匹配专项 rubric（数据库→rubric-database.md）     │
├─────────────────────────────────────────────────────┤
│ 第二步：扫描学生报告                                   │
│  ├── 列出文件夹内全部 .docx 文件                       │
│  ├── 按文件名规范解析（班级_姓名_学号）                │
│  └── 格式不符文件记入异常清单                          │
├─────────────────────────────────────────────────────┤
│ 第三步：逐一评价                                       │
│  对每篇报告：                                          │
│  ├── 读取报告全文（python-docx 提取文本/表格）         │
│  ├── 按三大题逐题核对检查点、打分                      │
│  ├── 汇总总分 → 映射五级制等级                        │
│  └── 生成 150-200 字综合评语（优点+不足+改进建议）     │
├─────────────────────────────────────────────────────┤
│ 第四步：生成成绩汇总 Excel                             │
│  ├── 调用 scripts/generate_grades_excel.py           │
│  ├── Sheet1 成绩明细（班级/学号/姓名/三题分/总分/等级/评语）│
│  ├── Sheet2 统计摘要（平均分/及格率/等级分布）         │
│  └── Sheet3 异常清单（文件名解析失败的文件）           │
├─────────────────────────────────────────────────────┤
│ 第五步：输出统计摘要                                   │
│  ├── 等级分布（优秀X人/良好X人/中等X人/...）           │
│  ├── 平均分、最高/最低分、及格率、优秀率               │
│  └── 常见问题 Top3（跨报告聚合共性问题）               │
└─────────────────────────────────────────────────────┘
```

### 步骤 4：查看输出

**输出 1：成绩汇总 Excel 文件**

文件名：`{课程名}_课设成绩汇总_{学期}.xlsx`

成绩明细表结构：

| 班级 | 学号 | 姓名 | 题目1-建模设计(50) | 题目2-管理优化(30) | 题目3-应用开发(20) | 总分 | 等级 | 综合评语 |

```bash

```

输入 JSON 格式见 `references/excel-output-spec.md`。

## 如何适应其他课程

本 Skill 已为多课程扩展做了设计准备。适应新课程只需 **3 步**：

### 第 1 步：创建该课程的专项量规

在 `references/` 下新建 `rubric-<课程>.md`，参照 `rubric-database.md` 的结构编写：

```markdown
# <课程名> — 课程设计评分专项量规

## 三大题评分标准

### 题目 1：xxx（XX 分）
> 固定结构说明...
#### 交付物检查点
- [ ] 检查点 1
- [ ] 检查点 2
#### 评分参考
| 得分 | 标准 |
...

### 题目 2：xxx（XX 分）
...

### 题目 3：xxx（XX 分）
...
```

**关键原则：**
- **三大题分值占比**：按该课程任务书的实际分值设定（不必是 50/30/20）
- **检查点清单**：从该课程的任务书中提取具体要求
- **固定 vs 任务相关**：标注哪些检查点每年不变、哪些随任务书微调

### 第 2 步：在 SKILL.md 中注册课程

编辑 `SKILL.md` 的 `supported_courses` 和"第一步：解析任务书"中的 rubric 匹配逻辑：

```yaml
supported_courses:
  - 数据库系统原理
  - 软件工程        # 新增
  - 软件测试与质量保证  # 新增
```

匹配逻辑（SKILL.md 第一步）：
```
数据库系统原理 → references/rubric-database.md
软件工程 → references/rubric-software-engineering.md   # 新增
软件测试 → references/rubric-software-testing.md       # 新增
```

### 第 3 步：使用时声明维度配置

Excel 生成脚本已**配置化**，无需改代码。批改时在评分 JSON 中声明该课程的维度：

```json
{
    "course": "软件工程",
    "dimensions": [
        {"key": "q1_requirement", "label": "题目1-需求分析(40)", "max": 40},
        {"key": "q2_design", "label": "题目2-系统设计(35)", "max": 35},
        {"key": "q3_testing", "label": "题目3-测试报告(25)", "max": 25}
    ],
    "grades": ["优秀", "良好", "中等", "及格", "不及格"],
    "students": [...]
}
```

> 脚本会根据 `dimensions` 自动生成对应的成绩表列，根据 `grades` 自动生成等级分布统计。

### 扩展示例：软件工程课设

假设软件工程课设三大题为：需求分析(40) + 系统设计(35) + 测试报告(25)

1. 创建 `references/rubric-software-engineering.md`：
   - 题目1 需求分析：用例图、需求规格说明书、需求追踪
   - 题目2 系统设计：类图、架构设计、数据库设计、接口设计
   - 题目3 测试报告：测试用例设计、缺陷分析、覆盖率

2. SKILL.md 注册课程

3. 批改时 JSON 声明 dimensions（如上示例）

**无需修改的部分（跨课程通用）：**
- `filename-conventions.md`：文件名规范所有课程一致
- `excel-output-spec.md`：Excel 输出规范通用（维度配置化）
- `generate_grades_excel.py`：脚本配置化，不改代码
- `rubric-general.md`：通用 5 维度量规作为参考基线

### 年度更新（同一课程，新一年任务书）

当同一课程新一年的任务书微调了题目内容时：

1. 将新任务书放入 `task/` 目录
2. 更新 `rubric-<课程>.md` 中**任务相关指标**部分（题目内容、检查点）
3. **固定部分无需改动**：三大题分值占比、考察要点、评分参考标准

详见 `rubric-database.md` 末尾的"每年任务书更新指引"。

## 常见问题

**Q: 学生报告可以是 PDF 吗？**

不可以。当前仅支持 `.docx` 格式，文件名规范也要求 `.docx` 扩展名。PDF 文件会被拒绝并记入异常清单。这已在文件名规范中强制要求。

**Q: 文件名格式不规范的文件怎么处理？**

格式不符的文件（如缺学号、段数不足、扩展名错误）会被拒绝评分，并记入 Excel 的"异常清单"工作表，注明异常原因。正常文件继续评分，不影响整体流程。

**Q: 任务书每年变化怎么办？**

任务书的**三大题分值占比和考察要点每年固定**，只需更新 `rubric-<课程>.md` 中的任务相关检查点（题目内容部分）。详见各 rubric 文件末尾的"每年任务书更新指引"。

**Q: 如何支持新的课程？**

按本文档[第 6 节](#如何适应其他课程)的 3 步操作：创建专项 rubric → 注册课程 → 声明维度配置。Excel 脚本无需改动。

**Q: 评语是如何生成的？**

每篇报告生成 150-200 字综合评语，包含：总体评价 + 三大题逐题评价（引用具体问题）+ 改进建议。评语要求具体、有针对性，避免泛泛而谈。

**Q: 能否批量重新生成 Excel 而不重新评分？**

可以。评分结果保存为 JSON 后，可反复运行 `generate_grades_excel.py` 重新生成 Excel，无需重新读取报告：

```bash
python scripts/generate_grades_excel.py --input grades.json --output 新成绩表.xlsx
```

**Q: 成绩表的维度列可以自定义吗？**

可以。维度在输入 JSON 的 `dimensions` 字段中声明，脚本自动适配。不同课程的三题名称和分值可以不同，脚本会根据配置生成对应的列。
