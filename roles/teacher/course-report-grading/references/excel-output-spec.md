# 成绩汇总 Excel 输出规范

## 文件命名

```
{课程名}_课设成绩汇总_{学期}.xlsx
```

示例：`数据库系统原理_课设成绩汇总_2024春.xlsx`

## 成绩表分值结构

> **忠实于任务书结构**：按三大题给分，而非通用5维度。
> 数据库课设三大题：题目1(50) + 题目2(30) + 题目3(20) = 100 分。
>
> 不同课程的三大题结构可能不同，脚本通过配置适配（见 `generate_grades_excel.py` 的 `DIMENSIONS` 常量）。

## 工作表结构

### Sheet 1：成绩明细（主表）

| 列 | 字段 | 类型 | 说明 |
|----|------|------|------|
| A | 班级 | 文本 | 从文件名解析 |
| B | 学号 | 文本 | 从文件名解析（保留前导零） |
| C | 姓名 | 文本 | 从文件名解析 |
| D | 题目1-建模设计(50) | 数字 | 数据库建模分析设计题得分 |
| E | 题目2-管理优化(30) | 数字 | 数据库管理和维护实验得分 |
| F | 题目3-应用开发(20) | 数字 | 数据库应用开发得分 |
| G | 总分 | 数字 | D~F 之和（0-100） |
| H | 等级 | 文本 | 优秀/良好/中等/及格/不及格 |
| I | 综合评语 | 文本 | 150-200 字评语 |

> **注**：上表为数据库课设的列结构。其他课程的三大题名称不同时，D/E/F 列标题相应调整，但"三题+总分+等级+评语"的结构不变。

### Sheet 2：统计摘要

| 统计项 | 值 |
|--------|-----|
| 应交人数 | N |
| 实评人数 | M |
| 缺交人数 | N-M |
| 平均分 | XX.X |
| 最高分 | XX |
| 最低分 | XX |
| 及格率(≥60) | XX% |
| 优秀率(≥90) | XX% |

等级分布表：

| 等级 | 人数 | 占比 |
|------|------|------|
| 优秀 | X | X% |
| 良好 | X | X% |
| 中等 | X | X% |
| 及格 | X | X% |
| 不及格 | X | X% |

### Sheet 3：异常清单（如有）

列出文件名解析失败、格式不符、内容异常的学生文件。

## 样式规范

### 表头

- 字体：微软雅黑 / 11pt / 加粗 / 白色
- 背景：深蓝色 `#2F5496`
- 居中对齐
- 冻结首行

### 等级着色（H 列）

| 等级 | 背景色 | 字体色 |
|------|--------|--------|
| 优秀 | `#C6EFCE`（浅绿） | `#006100` |
| 良好 | `#BDD7EE`（浅蓝） | `#1F4E79` |
| 中等 | `#FFEB9C`（浅黄） | `#7F6000` |
| 及格 | `#FCE4D6`（浅橙） | `#833C00` |
| 不及格 | `#FFC7CE`（浅红） | `#9C0006` |

### 总分列（G 列）

- 条件格式：`<60` 红色字体，`≥90` 绿色字体

### 列宽

| 列 | 宽度 |
|----|------|
| A-C | 12 |
| D-F | 16 |
| G | 8 |
| H | 8 |
| I | 60（评语较长） |

### 统计行（明细表末尾）

追加一行汇总：平均分（G 列）、各题平均分（D-F 列），浅灰背景 `#D9D9D9`。

## 生成方式

由 `scripts/generate_grades_excel.py` 脚本生成，依赖 `openpyxl`。
输入为评分结果 JSON 文件，输出为符合上述规范的 `.xlsx` 文件。

### 输入 JSON 格式

```json
{
    "course": "数据库系统原理",
    "semester": "2025-2026(1)",
    "dimensions": [
        {"key": "q1_modeling", "label": "题目1-建模设计(50)", "max": 50},
        {"key": "q2_management", "label": "题目2-管理优化(30)", "max": 30},
        {"key": "q3_development", "label": "题目3-应用开发(20)", "max": 20}
    ],
    "grades": ["优秀", "良好", "中等", "及格", "不及格"],
    "students": [
        {
            "class": "软件2301",
            "id": "120231080315",
            "name": "陈小鹏",
            "scores": {
                "q1_modeling": 45,
                "q2_management": 27,
                "q3_development": 18
            },
            "total": 90,
            "grade": "优秀",
            "comment": "..."
        }
    ],
    "anomalies": [
        {"filename": "xxx.docx", "reason": "文件名格式不符"}
    ]
}
```

> `dimensions` 和 `grades` 在 JSON 中声明，使脚本可适配不同课程的三题结构和等级名称。
