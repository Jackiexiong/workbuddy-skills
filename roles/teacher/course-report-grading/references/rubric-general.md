# 通用评分量规（5 维度基线）

> 本量规适用于所有计算机专业课程的课程设计 / 实验报告评分。
> 各课程可在 `rubric-<课程>.md` 中对"技术正确性"等维度做专项细化，
> 但 5 个维度的名称与分值保持固定，确保跨课程可比性。

## 评分维度总览

| 维度 | 分值 | 权重 | 评价核心 |
|------|------|------|---------|
| 任务完成度 | 30 | 30% | 是否覆盖任务书全部要求 |
| 技术正确性 | 30 | 30% | 代码 / 设计 / 实现是否正确 |
| 文档规范性 | 15 | 15% | 结构、格式、图表、参考文献 |
| 分析深度 | 15 | 15% | 实验分析、问题讨论、心得 |
| 创新拓展 | 10 | 10% | 超出基本要求的亮点 |

**合计：100 分**

---

## 维度 1：任务完成度（30 分）

### 评价要点

对照任务书要求，逐项核对是否完成：

- 任务书列明的全部功能点 / 内容项是否覆盖
- 必须提交的交付物是否齐全（设计文档、代码、测试报告等）
- 各项要求是否完整实现（而非部分 / 占位）

### 评分参考

| 得分 | 标准 |
|------|------|
| 27–30 | 全部要求完整覆盖，无遗漏 |
| 24–26 | 覆盖绝大部分要求，个别次要项缺失 |
| 21–23 | 完成主要要求，部分功能缺失或未实现 |
| 18–20 | 完成基本要求的一半左右 |
| <18 | 大量要求未完成 |

> **任务相关子项**：本维度具体检查点由 `rubric-<课程>.md` 结合任务书定义。

---

## 维度 2：技术正确性（30 分）

### 评价要点

- 代码 / SQL / 算法逻辑是否正确，能否运行
- 设计方案是否合理（架构、数据模型、流程）
- 是否存在明显技术错误（语法、逻辑、安全）
- 关键技术点的实现质量

### 评分参考

| 得分 | 标准 |
|------|------|
| 27–30 | 技术方案优秀，实现正确，无错误 |
| 24–26 | 方案合理，基本正确，个别小瑕疵 |
| 21–23 | 总体正确，存在若干明显问题 |
| 18–20 | 方案有缺陷，实现有多处错误 |
| <18 | 技术方案错误或无法运行 |

> **专项子指标**：本维度按课程细化为子指标，见 `rubric-<课程>.md`。

---

## 维度 3：文档规范性（15 分）

### 评价要点

- 报告结构完整（封面、目录、正文、结论、参考文献）
- 格式规范（标题层级、编号、字体字号统一）
- 图表清晰规范（有图题表题、编号连续）
- 代码块格式正确，有注释
- 参考文献格式规范

### 评分参考

| 得分 | 标准 |
|------|------|
| 14–15 | 结构完整，格式规范，图表精美 |
| 12–13 | 结构完整，格式基本规范，小瑕疵 |
| 10–11 | 结构有缺项或格式不统一 |
| 9 | 结构不完整，格式混乱 |
| <9 | 文档严重不规范 |

---

## 维度 4：分析深度（15 分）

### 评价要点

- 实验结果分析是否到位（不只是罗列结果）
- 问题讨论是否有深度（遇到的困难、解决方案）
- 测试结果解读是否充分
- 心得体会是否真实有反思

### 评分参考

| 得分 | 标准 |
|------|------|
| 14–15 | 分析深入，有独立思考和反思 |
| 12–13 | 分析到位，有一定深度 |
| 10–11 | 有分析但较浅，流于表面 |
| 9 | 分析缺失或照搬结论 |
| <9 | 无实质分析内容 |

---

## 维度 5：创新拓展（10 分）

### 评价要点

- 是否有超出基本要求的额外功能 / 优化
- 技术选型是否有亮点（用了更优方案）
- 解决问题的思路是否有创新
- 对课程知识的拓展应用

### 评分参考

| 得分 | 标准 |
|------|------|
| 9–10 | 有明显创新亮点或高质量拓展 |
| 7–8 | 有一定拓展，超出基本要求 |
| 5–6 | 完成基本要求，无明显拓展 |
| <5 | 仅完成最低要求 |

---

## 等级映射

| 等级 | 分数区间 | 含义 |
|------|---------|------|
| 优 | 90–100 | 全面完成且超出要求，有创新亮点 |
| 良 | 80–89 | 完成全部要求，质量良好 |
| 中 | 70–79 | 完成基本要求，部分环节有瑕疵 |
| 差 | 60–69 | 基本要求完成不全，存在明显问题 |
| 不及格 | <60 | 未完成核心要求或抄袭 |

## 评语撰写要求

每篇报告生成 100–200 字综合评语，结构：

1. **总体评价**（1-2 句）：完成情况概述 + 等级定性
2. **主要优点**（2-3 点）：结合维度具体说明
3. **不足与改进**（1-2 点）：指出问题并给改进方向
4. **鼓励 / 建议**（1 句）：针对该学生的具体建议

> 评语应具体、有针对性，避免泛泛而谈。引用报告中的具体内容佐证。
