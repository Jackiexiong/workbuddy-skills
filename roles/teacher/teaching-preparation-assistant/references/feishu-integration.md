# 飞书云文档集成指南

本指南介绍如何一次性配置飞书凭据，之后所有教学文档都能自动保存到指定的飞书文件夹。

---

## 重要说明：飞书 API 认证方式

飞书开放平台 API **不使用个人飞书账号密码**进行认证，而是使用**应用凭据**：

| 概念 | 说明 |
|------|------|
| App ID | 飞书应用的唯一标识，格式如 `cli_xxxxxxxxxxxx` |
| App Secret | 飞书应用的密钥，相当于应用的"密码" |
| folder_token | 飞书文件夹的标识，决定文档保存到哪个文件夹 |

这是飞书官方的安全设计——你创建一个"自建应用"，用应用的凭据调用 API，
而不是暴露个人账号密码。应用可以被管理员随时禁用或删除，比直接用账号密码更安全。

## 一次性配置流程（5 步）

以下配置只需做一次，之后所有文档自动使用此配置。

### 第 1 步：创建飞书自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/)，用你的飞书账号登录
2. 点击「开发者后台」→「创建企业自建应用」
3. 填写应用名称（如"教学文档助手"）和描述
4. 创建完成后，进入应用详情页

> **注意**：你需要是飞书租户的管理员，或有创建应用的权限。
> 如果你不是管理员，请联系管理员协助创建。

### 第 2 步：配置应用权限

在应用详情页的「权限管理」中，搜索并开通以下权限：

| 权限名称 | 权限标识 | 用途 |
|---------|---------|------|
| 查看、评论、编辑和管理云空间中所有文件 | `drive:drive` | 上传文件到云空间 |
| 上传文件 | `drive:file:upload` | 上传 Markdown 文件 |
| 查看、评论、编辑和管理文档 | `docx:document` | 创建和管理飞书文档 |

开通权限后，点击「创建版本」→「发布」（企业自建应用通常即时生效）。

### 第 3 步：获取应用凭据

在应用详情页的「凭证与基础信息」中，找到：

- **App ID**：格式为 `cli_xxxxxxxxxxxx`
- **App Secret**：一长串密钥字符串

请妥善保存这两个值。

### 第 4 步：获取目标文件夹的 token

决定教学文档保存到飞书的哪个文件夹，然后获取该文件夹的 token：

1. 打开飞书网页端 → 云文档
2. 进入你想存放教学文档的文件夹（如"教学资料"文件夹）
3. 查看浏览器地址栏，URL 类似：
   `https://xxx.feishu.cn/drive/folder/fldrXXXXXXXXXXXX`
4. URL 最后一段（`fldrXXXXXXXXXXXX`）就是 folder_token

> 如果不设置 folder_token，文档会创建在应用的默认空间中。
> 建议设置一个专用文件夹，方便管理和分享。

### 第 5 步：创建配置文件

运行以下命令，从模板创建配置文件：

```bash
python scripts/save_to_feishu.py --init
```

这会在 `scripts/` 目录下创建 `feishu_config.json` 文件。打开它，填入你的凭据：

```json
{
    "app_id": "cli_axxxxxxxxxxxx",
    "app_secret": "你的App Secret",
    "folder_token": "fldrXXXXXXXXXXXX"
}
```

> 只需修改 `app_id`、`app_secret`、`folder_token` 三个字段的值。
> 以 `_` 开头的字段是说明注释，不用管。

### 验证配置

填好凭据后，运行验证命令确认配置正确：

```bash
python scripts/save_to_feishu.py --check
```

如果输出"配置验证通过"，说明一切就绪，可以正常使用了。

如果验证失败，请根据错误提示检查：
- app_id / app_secret 是否填写正确
- 应用是否已发布
- 应用权限是否已开通

## 日常使用（配置完成后）

配置完成后，每次生成教学文档后只需一条命令：

```bash
python scripts/save_to_feishu.py --file "Python面向对象编程.md" --name "Python面向对象编程"
```

脚本会自动：
1. 读取已保存的配置（app_id、app_secret、folder_token）
2. 上传 Markdown 文件到飞书
3. 转换为飞书文档
4. 保存到你在配置中指定的文件夹
5. 返回飞书文档链接

运行成功后输出示例：

```

```

拿到链接后，在飞书中设置文档权限即可分享给学生。

### 命令参数说明

| 参数 | 说明 |
|------|------|
| `--file` | Markdown 文件路径（必填） |
| `--name` | 飞书文档标题（必填） |
| `--folder` | 临时指定文件夹 token（覆盖配置文件中的设置，可选） |
| `--config` | 指定其他配置文件路径（可选） |
| `--init` | 从模板创建配置文件 |
| `--check` | 验证配置是否正确 |

### 临时切换文件夹

如果你想把某次生成的文档保存到不同的文件夹，不需要修改配置文件，直接用 `--folder` 参数：

```bash
python scripts/save_to_feishu.py --file "实验指导.md" --name "实验指导" --folder fldrOtherFolderXXXX
```

## 手动导入替代方案

如果不想配置飞书 API（比如你不是管理员无法创建应用），也可以手动导入：

### 方法一：飞书云空间导入

1. 打开飞书云空间（飞书网页端 → 云文档）
2. 进入目标文件夹，点击右上角「+」→「导入」
3. 选择「Markdown」格式
4. 上传 `.md` 文件
5. 飞书会自动将 Markdown 转换为飞书文档

### 方法二：复制粘贴

1. 用 Markdown 编辑器（如 Typora）打开 `.md` 文件
2. 全选复制
3. 在飞书文档中粘贴
4. 飞书会自动识别 Markdown 格式（包括标题、代码块、表格等）

> **注意**：飞书的 Mermaid 图表渲染需要在新版飞书文档（docx）中才支持。
> 如果粘贴后 Mermaid 代码块没有渲染为图表，可能需要手动插入图表。

## 常见问题

### Q: 我不是飞书管理员，没法创建应用怎么办？

可以请管理员帮你创建应用并开通权限，然后把 App ID 和 App Secret 给你。
或者使用上面的「手动导入替代方案」，不需要 API 配置。

### Q: 配置文件保存在哪里？安全吗？

配置文件保存在 `scripts/feishu_config.json`，只存在于你的本地电脑上。
脚本不会将凭据上传到任何地方。建议：

- 不要把 `feishu_config.json` 提交到 Git 仓库（已在 .gitignore 中排除）
- 如果担心安全，可以将文件权限设置为仅自己可读

### Q: App Secret 忘了怎么办？

在飞书开放平台 → 应用详情 → 凭证与基础信息中，可以重置 App Secret。
重置后需要同步更新配置文件中的 `app_secret` 字段。

### Q: 导入报错 "permission denied" 怎么办？

确保：
1. 应用已开通所需权限（见第 2 步）
2. 应用已发布版本
3. 如果指定了 folder_token，确保应用有该文件夹的访问权限

### Q: 导入后 Mermaid 图表没有渲染怎么办？

飞书文档支持 Mermaid 代码块渲染。如果导入后图表没有显示：
1. 确认使用的是新版飞书文档（docx 格式）
2. 点击代码块右上角的语言选择器，确保设置为 "mermaid"
3. 如果仍未渲染，可以在飞书文档中手动插入「流程图」组件

### Q: 想换一个文件夹保存文档怎么办？

两种方式：
- **临时更换**：命令行加 `--folder` 参数，不影响配置文件
- **永久更换**：编辑 `feishu_config.json` 中的 `folder_token` 字段

### Q: 如何批量导入多个文档？

```bash
for %f in (*.md) do python scripts/save_to_feishu.py --file "%f" --name "%~nf"
```

或者让 AI 助手帮你批量执行。
